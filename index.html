Moving from prompt-and-copy to AI that plans, acts, and touches real systems.

Start with the room, not the tool. Ask: "Show of hands — who used an AI chat tool this week?" Then: "Who had it actually touch a file, a doc, or your inbox — not just reply in the chat box?" That gap is the talk.

Tier 1

From snippet to deliverable

Set up the pain before the demo: "How many of you have taken an AI reply and manually reformatted it into a Word doc or Excel sheet for a client?"

Tier 1 — Live demo

A messy requirement doc → a real file

  • Take a genuinely messy, anonymized client requirement doc — not a clean staged sample.
  • Ask for a structured .docx output: exec summary, grouped requirements, open questions, timeline table.
  • Open the actual file live. Point at one structural choice it made well.
Read this client requirement document and produce a structured
Word document with: an executive summary, a numbered list of
functional requirements grouped by category, a list of open
questions needing client clarification, and a rough timeline
estimate table. Save it as a .docx file I can send to the client.

"I didn't copy anything. That's the mental shift for tier one."

Narrate what's happening as it works, not just the end result. Full prompt variants (Excel scoping model, etc.) are in demo-scripts/tier1-deliverables.md.
35–60 min

Tier 2

Agentic workflows — the real unlock

This is the credibility segment — lean on real, specific war stories rather than hypotheticals.

Tier 2 — Live demo (pick one)

Multi-step work, not one prompt

Option A — RFP Assessment Agent

Read the RFP → extract every requirement → cross-check against the capability doc → produce a scored fit assessment with top gaps.

Option B — Inbox → Draft workflow

Search a live thread → summarize where things stand → draft a reply → create a follow-up doc with action items and owners.

"It can do things, not just tell me things."

Name the steps out loud before running it so the room can follow along. If using the inbox demo, be explicit about scope: read/summarize/draft only — nothing sends without review. Point at a flagged gap or uncertainty, not just the wins — that builds trust.
60–80 min

Tier 3

Coding agents

for the technical crowd specifically

"This next bit is for the engineers, but stay with me even if you're not — this is where the 'junior engineer' analogy really clicks."

Tier 3 — Live demo

Reading the codebase before touching it

  • Show a real failing test or messy function first, plainly — so the room can judge the before/after themselves.
  • Ask it to fix, refactor, or scaffold a new module following existing project conventions.
  • Narrate what it's reading, not just what it's writing — it checks other files before it changes anything.

"That's not autocomplete. That's delegation."

If code review comes up: agent output still goes through the same PR review process as anyone's code. Nothing merges without a human.

Full prompt variants (failing test / refactor / new module) are in demo-scripts/tier3-coding-agent.md.
80–90 min

Your turn

One task, your own laptop, right now

"Build me an Excel model from this messy CSV" — or — "Connect to my inbox and summarize unread mail."

Give the task before explaining tool mechanics — let people start immediately. If the room is slow, cue the seeded plant to share their result out loud first.

Closing

Three things to do this week

  • Request access to the tool(s) shown today.
  • Pick one workflow from today and try it on a real task this week.
  • Know where to ask for help — [name a person / channel here].
Close on next steps, not a recap. The room won't remember a summary slide — they'll remember what they're supposed to do by Friday.
