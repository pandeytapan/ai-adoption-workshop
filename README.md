# AI-First Adoption Workshop

A 90-minute session for Cynoteck to move the team from "AI = paste a paragraph, get a paragraph back" to **AI that plans, acts, and touches real systems**.

**Live slides:** [pandeytapan.github.io/ai-adoption-workshop](https://pandeytapan.github.io/ai-adoption-workshop/) — keyboard-navigable, no PowerPoint or Google Slides needed. Use → / ← or space to move between slides and elements, press `S` for speaker notes.

## Why this session exists

Most people at Cynoteck are using AI for snippets — pasting text into a chat box and copying the reply back out. That's maybe 10% of what's available, and it leaves 90% of the work still on the human. This workshop reframes AI around three escalating tiers of capability, each anchored in scenarios the team already recognizes: RFPs, client docs, code review, and client comms — not generic "write me a poem" demos.

## The arc, at a glance

1. **0–10 min — Reframe, not features.** Before/after: chat-box snippets vs. an agent that reads, checks sources, drafts, and flags what needs a human.
2. **10–35 min — Tier 1: From snippet to deliverable.** Turn a messy client requirement doc into an actual Word/Excel/PDF output, live.
3. **35–60 min — Tier 2: Agentic workflows.** Multi-step work — an RFP assessment agent, or a live inbox-to-draft-to-follow-up-doc workflow.
4. **60–80 min — Tier 3: Coding agents.** For the technical crowd — a live coding agent fixing a test or extending a module by reading the codebase first.
5. **80–90 min — Hands-on + close.** One live task on their own laptop, then 2–3 concrete next steps.
