# Speaker Notes — Talking Points by Segment

Use alongside [`01-session-outline.md`](01-session-outline.md). These are narrative cues, not a script to read verbatim — say them in your own voice.

## Opening (0–10 min)

- Start with the room, not the tool. Ask: "Show of hands — who used an AI chat tool this week?" (Most hands go up.) "Who had it actually touch a file, a doc, or your inbox — not just reply in the chat box?" (Fewer hands.) That gap *is* the talk.
- Land the line early and let it sit: *"If you're only using AI for snippets, you're using 10% of what's available and doing 90% of the work yourself."*
- Frame the three tiers as a ladder, not a feature list: **Tier 1 — it produces things. Tier 2 — it does things across systems. Tier 3 — it works inside your codebase like a teammate.** Tell the room they'll see all three live today.
- Don't demo anything yet. Resist the urge — the reframe needs to land on its own first.

## Tier 1 — Snippet to deliverable (10–35 min)

- Set up the pain before the demo: "How many of you have taken an AI reply and manually reformatted it into a Word doc or Excel sheet for a client?" That manual reformatting step is the 90% we're eliminating.
- Walk in with a genuinely messy input — a real (anonymized) client requirement doc, not a clean sample. Messy inputs build more trust than polished ones.
- Narrate what's happening as it works: "It's not just summarizing this — it's structuring it into sections, pulling out the actionable items, and it's about to hand me back an actual file I can attach to an email."
- When the file lands, open it. Don't just say it worked — show the doc, scroll through it, point at a specific place it made a good structural choice.
- Close the segment: "Notice what didn't happen — I didn't copy anything. That's the mental shift for tier one."

## Tier 2 — Agentic workflows (35–60 min)

- This is the credibility segment — lean on real, specific war stories rather than hypotheticals. Say what actually broke the first time you tried this and how it improved.
- Before running the RFP assessment agent, name the steps out loud first: "It's going to read the RFP, pull out every requirement, check each one against our capability doc, and score us — that's four steps most people would do by hand over an afternoon."
- If using the inbox demo instead: be transparent about what's connected and why ("this is read/summarize/draft only — nothing sends without my review").
- The key beat to land here: "It can do things, not just tell me things." Say this explicitly once the output appears — don't assume the room draws the conclusion themselves.
- Flag the human-in-the-loop moment deliberately: point at wherever the agent flagged something uncertain or needing review. This builds trust rather than fear.

## Tier 3 — Coding agents (60–80 min)

- Address the room shift directly: "This next bit is for the engineers, but stay with me even if you're not — this is where the 'junior engineer' analogy really clicks."
- Before the demo, show the failing test or the function to refactor first, plainly, so the room can judge the before/after themselves.
- As the agent works, narrate what it's reading, not just what it's writing: "Notice it's not autocompleting a line — it went and looked at three other files to understand our conventions before touching anything."
- Reframe explicitly at the end: "That's not autocomplete. That's delegation."
- If someone raises code quality or review concerns, welcome it — this is a genuine and reasonable question. Answer honestly: agent output still goes through the same PR review process as anyone's code.

## Hands-on + close (80–90 min)

- Give the task before explaining the tool mechanics — let people start immediately with whatever's already on their laptop.
- If the room is slow to start, cue the seeded plant to share their result out loud first — this is far more effective than a second explanation from the front.
- Close on next steps, not a recap. The room won't remember a summary slide; they'll remember what they're supposed to *do* by Friday.
- End with where to get help — a named person or channel, not "reach out to IT."
